# Nitro Type Research

## Overview
I chose Nitro Type as my reference game because it is one of the most popular and accessible typing games online. The gameplay is simple, competitive, and easy to understand, which makes it a strong reference point for our MVP.

## Key Features
- Real-time typing input
- Visual movement feedback
- Simple and addictive gameplay loop
- Short game sessions
- Progress based on typing accuracy and speed

## Ideas for Our Project
I think we can build a simplified version using only HTML, CSS, and JavaScript. Instead of copying the car racing idea exactly, we could use our own character/avatar. When the player types the code snippet correctly, the avatar moves forward.

This keeps the game simple while still giving visual feedback and making the experience more engaging than a normal typing test.

We could also create custom avatars based on each team member to make the project feel more personalized and unique to our group.

## Possible Improvements
Unlike Nitro Type, our game could focus more on learning coding syntax instead of only typing speed. We could introduce:
- HTML levels
- CSS levels
- JavaScript levels
- syntax-based challenges
- progressive difficulty

## MVP Takeaways
For the MVP, we should prioritize:
1. Simple gameplay
2. Immediate visual feedback
3. Beginner-friendly coding syntax
4. Mobile-friendly controls
